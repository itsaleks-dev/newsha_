Проект использует Vite + React + TypeScript и набор инструментов для разработки, тестирования и контроля качества кода.

⸻

Запуск проекта

Development режим

Запуск проекта в режиме разработки с горячей перезагрузкой:
• запускает Vite dev-сервер
• автоматически обновляет приложение при изменении кода

Используется для ежедневной разработки.

⸻

Production сборка

Сборка проекта для продакшена:
• выполняется проверка типов TypeScript,
• затем создаётся оптимизированная production-сборка через Vite.

Используется перед деплоем.

⸻

Preview production-сборки

Позволяет локально посмотреть production-версию приложения после сборки,
максимально близко к тому, как она будет работать на сервере.

Итог

Проект полностью готов к:
• активной разработке,
• масштабированию,
• CI/CD,
• поддержке строгого качества кода.

Скрипты покрывают весь цикл разработки:
от локального dev до production-сборки и тестирования.

В проекте используется Vite + React + TypeScript с разделённой конфигурацией TypeScript для разных сред:
приложение, тесты и конфигурационные файлы.

Такой подход:
• упрощает поддержку,
• повышает качество типизации,
• исключает конфликты между browser / node / test окружениями.

⸻

Vite configuration

vite.config.ts

Отвечает за:
• сборку приложения через Vite,
• подключение React,
• использование React Compiler (через Babel),
• настройку алиасов для удобных импортов.

Алиасы позволяют использовать абсолютные пути и соответствуют структуре проекта (app, shared, features, widgets и т.д.).

⸻

TypeScript конфигурации

В проекте используется несколько tsconfig-файлов, каждый из которых решает свою задачу.

Основной TypeScript (приложение)

Используется для:
• всего клиентского кода,
• React-компонентов,
• хуков, сервисов, стейта.

Особенности:
• строгая типизация (strict),
• поддержка JSX,
• совместимость с Vite (moduleResolution: bundler),
• отключён вывод файлов (TypeScript используется только для проверки типов),
• расширенные проверки качества кода (неиспользуемые переменные, строгие optional-типы и т.д.).

Алиасы синхронизированы с Vite.

⸻

TypeScript для тестов

Используется только для тестовых файлов.

Назначение:
• корректная типизация describe, it, expect и других функций Vitest,
• отсутствие конфликтов с основной конфигурацией приложения.

Проверяются исключительно файлы тестов, код приложения сюда не входит.

TypeScript для Node / конфигов

Используется для:
• vite.config.ts,
• vitest.config.ts,
• других конфигурационных файлов.

Отличия:
• работает в Node.js окружении,
• использует типы node,
• алиасы указывают на папку src из корня проекта.

Алиасы путей

Во всех конфигурациях используется единая система алиасов:
• @ — корень src
• @app, @shared, @entities, @features, @widgets, @pages, @types

Это:
• улучшает читаемость импортов,
• уменьшает связанность файлов,
• соответствует Feature-Sliced архитектуре.
